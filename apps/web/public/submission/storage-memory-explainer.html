Where The Evidence Lives

The app does not pretend everything is inside one database. Each layer has a job.

PROOF MAP

Walrus Blob

  • Stores immutable payloads and media.
  • Team profiles, gallery art, output JSON.
  • Returns blob IDs and certified object links.

Walrus Memory

  • Stores recallable knowledge for Gil.
  • Global schedule, teams, players, roast traits.
  • User notebook per Sui wallet address.

Sui Objects

  • Stores public receipts and gates.
  • Predictions, scoreboard, match registry.
  • OutputRecord stores blob pointer + content hash.

One output, three proofs

User actionWallet signs the action or session.
→
Blob / MemoryPayload or recall context is stored on Walrus.
→
Sui receiptObject stores pointer, hash, gate, or score.
→
TrackingUI exposes links for judges to inspect.

Important: a Walrus blob is not “inside” Sui. Sui stores the receipt: object ID, blob ID, content hash, score, prediction gate, or pointer. Walrus stores the data.

How Gil Remembers

Static tournament knowledge is global. User shame is wallet-scoped. Both are recallable.

MEMORY MAP

Static / Global Memory

shared by everyone
  • World Cup 2026 fixture schedule.
  • Groups, venues, kickoff locks, result updates.
  • Team profiles: coach, flag, squad list.
  • Player roast traits for famous football habits.
  • Updated when results, postponements, or knockout fixtures change.

User / Wallet Memory

one namespace per wallet
  • Chat history and favorite teams.
  • Prediction behavior and previous bad takes.
  • Roasts Gil already delivered.
  • Notebook recall for the connected Sui address.
  • Used to make day-N Gil more personal than day-1 Gil.

Signed action lifecycle

1. ConnectSui wallet proves identity.
2. SignPrediction or output action is wallet-gated.
3. StoreWalrus Memory or Blob keeps the data.
4. AnchorSui object records hash, pointer, or score.
5. RoastGil recalls it later and cites the mess.

Prediction actions sign a Sui transaction. Chat/roast/notebook actions require verified wallet session; important generated outputs can be stored as Walrus blob payloads and anchored by Sui OutputRecord.
