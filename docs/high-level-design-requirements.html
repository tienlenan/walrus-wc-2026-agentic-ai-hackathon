High-Level System Design

The Daily Walrus is a wallet-gated AI football app where every important action gets a signed Sui proof, Walrus stores the durable payloads, and Walrus Memory gives Gil recall across sessions.

Walrus Memory

Walrus Blob

Immutable media and JSON payloads: posters, gallery art, team profiles, output records.

Sui Objects

Owned prediction gates, result settlement, score records, and blob/hash receipts.

Index Cache

Supabase indexes fixtures, leaderboard, proof URLs, and rebuildable UI state.

Wallet Boundary

No anonymous writes. Prediction, roast, notebook, and share outputs start from a connected wallet session.

Memory Boundary

Walrus Memory is the source of truth for Gil's recall. Supabase can be rebuilt from proofs and blobs.

Contract Boundary

Sui stores compact proofs: object IDs, blob IDs, content hashes, gates, scores, and settlement state.

Judge Boundary

Runtime Tracking exposes live network, package, object, memory, blob, and explorer links.

Design rule: store big content on Walrus, store recall facts in Walrus Memory, store public receipts and gates on Sui. The app can be fast without pretending the cache is the source of truth.

Signed Action + Data Flow

Four flows cover the product: prediction, chat/roast output, global memory refresh, and judge verification. Each flow has one clear owner and one clear proof surface.

Prediction

1. Wallet signs

User selects an open match. UI checks kickoff gate before asking for signature.

2. Sui object

Prediction object stores match ID, pick hash, owner, and lock state.

3. Gil remembers

Memory records the take so future roasts can cite the user's football crimes.

4. Settled result

Oracle seed updates final score, closes gate, scores prediction, updates board.

Chat + Roast

1. Recall context

Agent loads global schedule, team facts, player traits, and wallet notebook.

2. Generate answer

Gil answers fixtures, team profile, prediction history, or roast prompt.

3. Store output

Important generated payloads are written to Walrus Blob when publisher is configured.

4. Anchor proof

Sui OutputRecord stores blob pointer and content hash for verification.

Global Memory

1. Fixture seed

World Cup schedule, groups, teams, venues, and flags are stored as global memory.

2. Match changes

Postponed fixtures, knockout pairings, and settled results update the same namespace.

3. Team profiles

Coach, squad, flag, and player roast traits are stored as reusable recall facts.

4. App reads once

Gil answers schedule/team questions from memory instead of hardcoded page text.

Tracking

1. Runtime config

Network, package ID, registry object, memory namespace, and publisher status.

2. Object links

Sui Explorer links point to owned predictions, output records, and settlement objects.

3. Walrus links

Blob and site links point to the exact stored media, JSON, and static app artifacts.

4. Rebuild check

Cache/index can be regenerated from Walrus and Sui proof surfaces.

System Components

  • Web SPA: prediction desk, chat, gallery, team profiles, tracking pages.
  • Agent API: Mastra tools for fixtures, memory recall, roasts, and output records.
  • Sui package: prediction registry, lock/settle rules, score and output proof objects.
  • Walrus: blobs for media/payloads, Walrus Memory for static and wallet-scoped recall.

Requirement Fit

  • Persistent AI memory: Walrus Memory, split by global and wallet namespace.
  • On-chain verifiability: Sui object receipts link to blob IDs and content hashes.
  • Prediction integrity: match gates close before kickoff and settle after results.
  • Submission proof: Runtime Tracking lists deployed contract, object, memory, and site links.
