High-Level Design / Submission Pack

The Daily Walrus

A wallet-gated World Cup 2026 prediction and roast app where Walrus Memory stores what Gil remembers, Sui objects prove what happened, and Supabase remains a rebuildable index.

testnet verified -> mainnet ready after funding

Product Goal

Make predictions, ask fixture questions, and receive personalized roasts that improve over time because memory is persistent and wallet-scoped.

Proof Goal

Every important output should be traceable through Walrus Memory, Walrus blobs, Sui objects, or a rebuildable runtime index.

Demo Goal

Judges can connect a wallet, submit a prediction, inspect memory, see leaderboard scoring, and open contract/blob links from Tracking.

Prediction Gate

  • Match must be registered on-chain.
  • Kickoff time must be in the future.
  • Settled matches are permanently closed.

Memory Gate

  • Personalized reads require verified wallet session.
  • Global fixture memory is public app context.
  • Output pointers are tracked for audit.

Mainnet Gate

  • Publish Move package to Sui mainnet.
  • Publish site/assets to Walrus mainnet.
  • Replace testnet IDs in tracking and submission.

Communication Flows

Runtime Paths Judges Can Verify

wallet -> memory -> proof

Prediction and Settlement

Prediction
1. Load fixturesWeb calls `/api/game/snapshot` and receives on-chain gate state.
2. Sign txUser signs `submit_prediction` against current Sui network.
3. Index eventServer indexer mirrors Sui events into Supabase.
4. Result seedOracle scores after result input and can settle match.
5. LeaderboardScoreboard and Shameboard update from indexed result.

Chat / Roast / Notebook

Memory
1. Verify walletSession token anchors the resource ID to Sui address.
2. RecallGil retrieves user, fixture, team, and player roast memory.
3. GenerateAI runtime applies language, severity, persona, and context.
4. StoreImportant output is written to memory and/or blob proof.
5. RegisterOutputRecord can point to hash/blob for public audit.

Tracking Page

Audit
1. NetworkShows active Sui network and contract IDs.
2. MemoryShows global/team/player memory sync receipts.
3. WalrusShows blob counts and openable aggregator URLs.
4. FixturesShows total, registered, open, closed, and not-on-chain counts.
5. SourcesLinks back to source schedule and squad references.
Browser test requirement: the app now exposes the current Sui network and checks wallet gas, so testnet wallets fail with an actionable faucet message instead of an opaque wallet popup error.
